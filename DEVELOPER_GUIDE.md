## 1. ما هذا النظام؟

**بوابة عمليات مستودعات** تعمل فعليًّا في شركة برندزو ببنغازي (ليبيا) — عربية RTL بالكامل، بالدينار الليبي.
ليست عرضًا توضيحيًّا: موظّفون يمسحون باركودات ويصدرون مستندات مرقّمة رسميًّا كل يوم.

**تعمل على ثلاث منصّات من مصدر واحد:** موقع ويب · تطبيق PWA يُثبَّت ويعمل دون إنترنت · برنامج ويندوز (قشرة Tauri).

| | |
|---|---|
| **الحيّ** | https://albarshi996.github.io/warehouse-system/ |
| **المستودع** | `albarshi996/warehouse-system` — فرع `main` ينشر تلقائيًّا |
| **الواجهة** | Astro 6 (توليد ساكن) + جزر React + Tailwind 4 |
| **البيانات** | Firebase: Auth (بريد/كلمة مرور) + Firestore (خطة Spark المجانية) |
| **الباك اند** | موديول أودو حقيقي `brandzo-addons/brandzo_warehouse` — 10/12 مرحلة، **غير مربوط بالواجهة بعد** |

## 2. التشغيل في ثلاثة أوامر

```bash
npm install
npm run dev      # http://localhost:4321/warehouse-system/
```

**للدخول تحتاج حسابًا في Firebase** — كل صفحات `/dashboard/**` محميّة. اطلب من المالك إنشاء حساب لك وإسناد دور، أو أنشئ مشروع Firebase خاصًّا بك واملأ `.env`:

```
PUBLIC_FIREBASE_API_KEY=...
PUBLIC_FIREBASE_AUTH_DOMAIN=...
PUBLIC_FIREBASE_PROJECT_ID=...
PUBLIC_FIREBASE_STORAGE_BUCKET=...
PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
PUBLIC_FIREBASE_APP_ID=...
```

### الأوامر التي تهمّك

| الأمر | ماذا يفعل | متى |
|---|---|---|
| `npm test` | 252 اختبار وحدة (Node test runner، بلا مُشغّل خارجي) | **قبل كل commit** |
| `npm run audit` | تدقيق البنية: صفحات يتيمة · روابط مكسورة · مجموعات Firestore بلا قاعدة · تسريب صلاحيات | **قبل كل commit** |
| `npm run build` | بناء الإنتاج (40 صفحة) + توليد `sw.js` | قبل الدفع |
| `npm run preview` | يخدم **البناء** لا المصدر — ⚠️ استخدمه لفحص جزر React (انظر §8) | عند التحقّق البصري |
| `npx eslint .` | **خطّ الأساس 122 مشكلة** (54 خطأ + 68 تحذيرًا) سابقة — احكم بالجديد لا بالمجموع | قبل الدفع |

`npm test` و`npm run audit` **حاجزان صلبان في خط النشر**: فشلٌ في أيّهما ⇒ لا بناء ولا نشر.

## 3. المعمارية — القاعدة الحاكمة

```
src/services/**     ← منطق خالص، بلا DOM ولا Firebase في دوال الحساب.  هنا الاختبارات.
src/components/**   ← جزر React تُرطَّب في المتصفّح (client:load).
src/pages/**        ← صفحات Astro. رقيقة: تركّب ما سبق.
src/layouts/        ← DashboardLayout.astro — القشرة والحارس والقائمة.
```

> ### 🔑 القاعدة الذهبية للكود
> **المنطق القابل للاختبار لا يُكتب داخل `<script is:inline>` في صفحة Astro.**
> الصفحات محميّة بالدخول، فلا يمكن فحصها آليًّا؛ وما يعيش داخل وسم `<script>` لا يمكن استيراده في اختبار. ضع القرار في `services/` واستدعِه.
>
> **مخالفات قائمة (دَين تقني معروف):** `stock-operations.astro` و`assets-inventory.astro` و`erp-workflows.astro` تحمل منطقًا ضخمًا مضمّنًا — وهي أيضًا أكبر مصادر مشاكل اللينت. لا تُضِف إليها؛ استخرج عند التعديل.

### أنماط ثابتة في الخدمات
- **معرّفات حتميّة:** `vehicles/{اللوحة}` · `assets/{الكود}` · `preparatory_meetings/{M01..M07}` — إعادة الرفع تُحدّث ولا تُكرّر.
- **سجلّات append-only:** `scans` · `audit` · `events` — لا تعديل ولا حذف (تفرضه قواعد الخادم).
- **الترقيم الرسمي بمعاملة:** عدّاد في `counters/` داخل `runTransaction` — الرقم يُحجز مرّة ولا يُحرق ثانية. الصيغة `GRN-2026-0001` · `TRIP-2026-0001` · `MOM-2026-0001`…
- **آلات حالات صريحة:** لا قفز بين الحالات؛ الانتقال المسموح مُعرَّف في جدول (`states.js` · `MEETING_TRANSITIONS`…).

## 4. نظام الوصول — ثلاث طبقات

```
navCatalog.js   ← المصدر الواحد: أي صفحة، في أي مجموعة، لأي أدوار
      ├─→ DashboardLayout.astro   يرسم القائمة
      ├─→ pageAccess.js → AuthGate.jsx   يمنع فتح الشاشة
      └─→ scripts/audit-portal.mjs   يكشف الانحراف

firestore.rules ← الإلزام الحقيقي على الخادم: يمنع البيانات لا الشاشة
```

**لماذا هذا التصميم؟** حتى 23.07.2026 كانت بنية القائمة مكتوبة داخل `DashboardLayout.astro`، فلم يستطع الحارس قراءتها، فكُتبت قائمة صلاحيات ثانية يدويًّا — **وانحرفت**: الحارس كان يقيّد دورين فقط بينما القائمة تُخفي روابط عشرة أدوار أخرى تفتح صفحاتها بكتابة الرابط. الآن الاثنان يقرآن من ملف واحد، واختبار انحراف يمنع عودة الفجوة.

### لإضافة صفحة جديدة
1. أنشئ `src/pages/dashboard/<اسمها>.astro` بـ`<DashboardLayout>`.
2. **أضِفها إلى `NAV_GROUPS` في [`src/services/auth/navCatalog.js`](src/services/auth/navCatalog.js)** — مجموعةً و`roles` إن كانت حسّاسة.
3. `npm run audit` — يفشل إن نسيت الخطوة 2 (لا صفحة يتيمة).
4. إن كانت تكتب في Firestore: أضِف قاعدة في `firestore.rules` — التدقيق يفشل بدونها.

**الأدوار الاثنا عشر** في [`roles.js`](src/services/auth/roles.js)، وخريطة الدور→المجموعات في [`navAccess.js`](src/services/auth/navAccess.js). الأدمن يرى كل شيء؛ **الدور المجهول يسقط على `viewer` (أقل صلاحية) لا على «بلا قيد»**.

## 5. محرّك المستندات — نموذج بملف مخطّط واحد

النماذج الورقية الـ21 لم تُحوَّل إلى 21 شاشة، بل إلى **محرّك واحد يقرأ مخطّطًا**:

```
services/documents/
  schemas/grn.js        ← الحقول · الأعمدة · القواعد · من يعتمد   ← أضِف ملفًّا مثله
  schemas/index.js      ← السجلّ
  numbering.js          ← حجز الرقم بمعاملة
  states.js             ← آلة الحالات ومن يملك كل نقلة
  documentsService.js   ← حفظ · نقلات · سجلّ تدقيق · مستمعات
```

**إضافة نموذج جديد = ملف مخطّط، لا شاشة.** الحقول والجداول والطباعة والترقيم والاعتماد بالدور: كلها من المحرّك.

**الحالة:** `GRN` كامل ومُجرَّب حيًّا. المتبقّي `F2` (سلسلة الشراء وتفعيل `links` للمطابقة الثلاثية) فما بعدها — [`ROADMAP.md`](ROADMAP.md) §11.5.

## 6. الحرّاس الستة (قواعد العمل غير القابلة للتفاوض)

مفروضة في موديول أودو بـ`@api.constrains`، ومرآتها في الواجهة وقواعد Firestore:

1. **لا استلام «منجَز» قبل اعتماد مفتّش الجودة.** 🥇
2. **لا خروج بضاعة بلا تصريح بوابة معتمد.** 🏅
3. **لا صرف يخالف FEFO** (الأقرب انتهاءً أولًا).
4. **لا ترحيل فاتورة قبل المطابقة الثلاثية** (أمر شراء ↔ استلام ↔ فاتورة). ⚖️
5. **لا تسوية جرد بلا اعتماد.** 🔒
6. **لا كتابة على فترة محاسبية مقفلة.**

عند تعديل أي مسار يمسّ هذه: الحارس يُفرض في **ثلاث طبقات** (واجهة · خدمة · قواعد خادم). لا تكتفِ بالأولى.

## 8. فخاخ موثّقة (وقعنا فيها فعلًا)

- **`astro preview` لا `astro dev` لفحص جزر React** — لخلل ترطيب `createRoot` في وضع dev. افحص على البناء.
- **كاش GitHub Pages عشر دقائق** بعد النشر — إن لم تظهر صفحتك: `Ctrl+Shift+R`.
- **`global.css` يسحق أيقونات FontAwesome بـ`!important`** — انتبه عند إضافة أنماط عامة.
- **`BASE_URL` في Astro 6 قد يأتي بلا شرطة ختامية** — طبّعه دائمًا قبل التركيب، وإلا التصقت الروابط.
- **المسارات العربية تصل من `location.pathname` مُرمَّزة** — `decodeURIComponent` قبل المقارنة.
- **الباركود:** الأصفار البادئة تُسقط للأرقام الخالصة فقط (`00251` = `251`)، وغير الرقمي (`IP34927`) لا يُمسّ. نقطة التطبيع واحدة: `normalizeBarcode`.
- **الرصيد لا تحرّكه المستندات بعد** — مصدره الشيت المستورد. لا تَعِد المستخدم بغير ذلك في الواجهة.

## 12. قواعد التعامل مع هذا المستودع

1. **صفر حذف.** الميزات لا تُحذف بل تُنقل أو تُربط. النماذج الورقية في `public/forms/` تبقى كما هي.
2. **تحقّق قبل الثقة.** لا تصدّق ادّعاء وثيقة قبل مطابقته بالكود — وثائقنا انحرفت من قبل، ولهذا وُجد `npm run audit`.
3. **الواجهة عربية RTL دائمًا.** الكود والالتزامات إنجليزية/مختلطة، لكن كل ما يراه المستخدم عربي.
4. **التوقيعان الرسميان** (`public/identity/signatures/`) يُدرجان تلقائيًّا في المحاضر والخطابات — لا تُغيّر مسارهما.
5. **لا تُدخل أسرارًا في المستودع.** مفاتيح Firebase عبر `.env` محليًّا وGitHub Secrets في النشر.
