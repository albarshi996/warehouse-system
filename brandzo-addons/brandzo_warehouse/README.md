# brandzo_warehouse — تقرير الموقف التشغيلي والبرمجي

> موديول Odoo 19 يجسّد الدورة المستندية اللوجستية لـ Brandzo (12 مرحلة / 21 نموذجاً) وقواعدها
> الذهبية كحُرّاس صارمين. هذا الملف **تقرير موقف للمراجعة** — يوثّق ما أُنجز، وما تبقّى، وبروتوكول
> العمل، والاكتشافات المعمارية. الخطة المرجعية الكاملة في
> [`ODOO_BACKEND_DEVELOPMENT_PLAN.md`](../../ODOO_BACKEND_DEVELOPMENT_PLAN.md) بجذر المستودع.

**الحالة:** S1→S6 + S5a + S5b كلها على `main` — القواعد الذهبية الأربع + المرتجعات/الإتلاف/الإشعار الدائن + الجرد الدوري وحارس التسويات. المتبقي: S12 الإغلاق المالي (+ التعميقات الاختيارية).
**المقاييس:** 35 ملفاً · 10 كائنات (توسعة 7 قياسية + 3 نماذج مخصصة) · 7 مجموعات أمان · 42 حالة اختبار في 10 ملفات.

## 4) التقييم المعماري — اكتشافات Odoo 19 وكيف تغلّبنا عليها

| الاكتشاف (من فحص المصدر الفعلي) | الأثر | حلّنا البرمجي |
|---|---|---|
| **نسخة Community**: لا `quality`/`quality_control` (مؤسسي) | لا فحص جودة أصلي | بنينا QC كحقول (`bz_qc_state` وغيرها) + حارس مزدوج على `stock.picking` |
| لا `account_3way_match` (مؤسسي) | لا مطابقة ثلاثية أصلية | بنيناها فوق `is_purchase_matched` و`purchase_line_id` كحارس على `_post` |
| 🔴 **`stock.valuation.layer` أُزيل في v19** | تغيّر بنية التقييم | نستهدف التقييم على `stock.move` (`value`, `remaining_qty`) — مهم لتسويات S5 |
| `purchase_requisition` و`product_expiry` **موجودان** | لا داعي لإعادة الاختراع | بنينا PR وFEFO **فوقهما** (FEFO = ترتيب `removal_date, in_date, id`) |
| `qty_done` أُعيدت تسميته إلى **`quantity`** على `stock.move.line` | كسر توافق قديم | كل كودنا يستخدم `quantity` |
| لا مفسّر Python على جهاز التطوير | تعذّر `py_compile`/تثبيت محلي | **تحقّق ساكن**: سلامة XML عبر مُحلّل .NET · تأكيد مرابط الوراثة بأرقام أسطر حقيقية · فحص الترميز (BOM/محارف) |

### أنماط هندسية اعتمدناها
- **حارس مزدوج الطبقة:** `button_validate`/`_post` (تفاعلي، رسالة واضحة) + `@api.constrains` (صارم لا يُتجاوز برمجياً).
- **`post_init_hook`:** يمنح السجلات التاريخية القيمة الآمنة (اعتماد/اجتياز) حتى لا تُحجَب بعد التثبيت.
- **تكامل بلا تكرار:** استهلاك عرض تواريخ الصلاحية من `product_expiry` والاكتفاء بشارة تلخيصية على مستوى المستند.
- **نقاط ربط مؤكَّدة:** `button_validate` (`stock_picking.py:1399`) · `_post` (`account_move.py:5526`) · `purchase_line_id` (`purchase/…/account_invoice.py:528`).
