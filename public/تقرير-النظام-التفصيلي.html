02

قواعد البيانات

المشروع يستخدم أربعة أنواع تخزين لأغراض مختلفة. الفعّال حاليًّا هو Firebase (Firestore + Auth) والتخزين المحلي؛ وPostgreSQL داخل أودو ينتظر بيئة التشغيل.

🔥 Firebase Firestore — قاعدة السحابة الأساسية

مشروع brandzo-erp-2026 على خطة Spark المجانية. قاعدة بيانات NoSQL تعمل بنمط offline-first (تعمل دون إنترنت وتزامن عند العودة). المجموعات المستخدمة فعليًّا في الكود:

🔒 قواعد أمان Firestore — نُشرت 2026-07-15 كشف اختبارٌ مباشر أن قاعدة البيانات كانت مفتوحة للعالم: أي شخص بلا حساب يقرأ ويكتب ويحذف، بل ويرقّي نفسه إلى مدير عام. الآن يحكمها ملف firestore.rules (107 أسطر): لا وصول بلا تسجيل دخول · الأدوار لا يكتبها إلا المدير العام · سجلّ المسح غير قابل للتعديل أو الحذف · منع افتراضي لكل ما لم يُذكر. أُعيد الاختبار: كل المحاولات PERMISSION_DENIED.

🔐 Firebase Authentication — الهويّة والدخول

مزوّد Email/Password مُفعّل. كل حساب له User UID فريد، ويُربط بمستند في مجموعة users لتحديد دوره. المدير العام يُمنح صلاحية الأدمن تلقائيًّا عبر تمهيد بالبريد.

💽 التخزين المحلي (localStorage)

يُخزّن تفضيلات المتصفّح دون سحابة:

csp_units_v2 / v3

dashboard cards

📊 طبقة Excel — تبادل البيانات

عبر مكتبة xlsx: استيراد/تصدير الأصناف والسجلّات بأسماء حقول موحّدة (excelSchema.js) تُطابق أسماء أودو — فيتحوّل الملف من/إلى نفس بنية Firestore.

🐘 PostgreSQL — قاعدة أودو (الباك اند)

لماذا لم تُفعّل بعد؟ موديول brandzo_warehouse يحتاج خادم أودو حيًّا (Python + PostgreSQL). لا يوجد مفسّر Python على جهاز التطوير، لذا تحقّقنا من الموديول ساكنًا فقط. تشغيله الحيّ = المرحلة M2 (Oracle Cloud المجاني).

نظام الوصول والأدوار

أحدث ما بُني: نظام دخول كامل يربط من يدخل بـما يراه وبـما يفعله — على أربع طبقات.

1️⃣ الدخول

شاشة /login عربية بـ Firebase Auth. جلسة محفوظة بين الزيارات.

2️⃣ الحارس

AuthGate يحمي كل صفحات لوحة التحكم؛ غير المسجّل يُحوّل للدخول قبل ظهور المحتوى.

3️⃣ القائمة حسب الدور

RoleNav يُخفي المجموعات/الروابط التي لا تخصّ الدور. المصدر: navAccess.js.

4️⃣ إدارة المستخدمين

صفحة أدمن تُسند الأدوار العشرة وتفعّل/تعطّل الحسابات عبر Firestore.
