## ما الذي يفعله

نظامٌ واحد يغطّي دورة المستودع من الاستلام إلى الخروج، ويُشغَّل من **ثلاث منصّات بمصدر واحد**: موقع ويب · تطبيق يُثبَّت على الجوال والكمبيوتر **ويعمل دون إنترنت** (PWA) · برنامج تنصيب ويندوز.

### الأنظمة الحيّة

| النظام | ماذا يقدّم |
|---|---|
| 📑 **محرّك المستندات** | مذكرات الاستلام (GRN) وأخواتها: ترقيم رسمي متسلسل · اعتماد بالدور · سجلّ تدقيق دائم · طباعة بالهوية |
| 📦 **ماستر الأصناف** | مصدر أصناف واحد للبوابة كلّها: استيراد بالإكسل · باركودات متعدّدة للصنف · أرصدة بتشغيلات وصلاحيات (FEFO) |
| 📡 **العمليات المخزنية** | مسح باركود من الجوال (يعمل على آيفون) · سجلّ مسح سحابيّ لا يُعدَّل · شاشة متابعة لحظية للمدير |
| 🚚 **سلاسل الإمداد والحركة** | دورة الرحلة من الإرسال إلى إثبات التسليم بتوقيع إلكتروني · تصاريح بوابة · رقابة وقود · مركز صيانة بأوامر شغل · عُهد عينية |
| 📊 **لوحة القيادة التنفيذية** | ثمانية مؤشرات لحظية: الأسطول · POD · OTIF · انحراف الوقود · أوامر الشغل · MTTR · العُهد · قرب انتهاء الصلاحيات |
| 🏛️ **الهيكل التنظيمي** | مصدر واحد يولّد التقارير والكتالوج · تحرير تفاعليّ بإصدارات · طباعة رسمية |
| 🤝 **الاجتماعات التحضيرية** | أجندة وعرض ومحضر حيّ · محاضر وخطابات مرقّمة رسميًّا بالتوقيعات المعتمدة |
| 🧑‍💼 **التوظيف الذكي** | كتالوج الوظائف · خطّ أنابيب المرشحين · السيرة داخل قاعدة البيانات |
| 🚗 **جرد المركبات** | سجلّ أسطول سحابيّ: فحوص مرقّمة · صيانة · زيوت · حوادث |
| 🎓 **محاكي أودو** | تدريب على الدورة المستندية الكاملة بفرض القواعد الذهبية آليًّا |

### تحت الغطاء

- **الوصول:** اثنا عشر دورًا، لكلٍّ ما يخصّه — تُشتقّ صلاحية كل صفحة من **مصدر واحد** يقرأه الحارس والقائمة والتدقيق معًا، والإلزام الحقيقي في قواعد Firestore على الخادم.
- **الحرّاس الستة:** قواعد عمل لا تُخترق — لا استلام «منجَز» بلا اعتماد جودة · لا خروج بلا تصريح بوابة · FEFO · المطابقة الثلاثية قبل الترحيل · لا تسوية جرد بلا اعتماد · لا كتابة على فترة مقفلة.
- **باك اند أودو:** موديول حقيقي `brandzo_warehouse` (10 من 12 مرحلة) — جاهز وغير مربوط بالواجهة بعد.

## التقنيات

Astro 6 (توليد ساكن) · React (جزر تفاعلية) · Tailwind 4 · Firebase (Auth + Firestore) · Tauri (برنامج ويندوز) · GitHub Pages + Actions

## التشغيل

```bash
npm install
npm run dev
```

ثم افتح `http://localhost:4321/warehouse-system/`. **كل صفحات لوحة التحكم محميّة بالدخول** — تحتاج حسابًا في Firebase ومتغيّرات `PUBLIC_FIREBASE_*` في ملف `.env` (التفاصيل في دليل المطوّر).

| الأمر | الغرض |
|---|---|
| `npm test` | 252 اختبار وحدة |
| `npm run audit` | تدقيق البنية: صفحات يتيمة · روابط مكسورة · قواعد أمان · تسريب صلاحيات |
| `npm run build` | بناء الإنتاج (40 صفحة) |
| `npm run preview` | معاينة البناء — استخدمه لا `dev` عند فحص واجهات React |

الدفع إلى `main` يمرّ ببوابة جودة (اختبارات + تدقيق) قبل البناء والنشر التلقائي.

## البنية

```
src/
  services/     ← منطق خالص مُختبَر (هنا يُكتب القرار)
  components/   ← جزر React
  pages/        ← صفحات Astro (رقيقة)
  layouts/      ← القشرة والحارس والقائمة
public/         ← أصول ثابتة · النماذج الورقية · التقارير HTML
brandzo-addons/ ← موديول أودو الحقيقي
scripts/        ← مولّدات ومدقّقات
docs/           ← مراجع أودو (وarchive/ للوثائق التاريخية)
```
